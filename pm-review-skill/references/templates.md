# 评审报告模板

## 标准评审报告模板

```markdown
# 页面评审报告

## 基本信息
- **评审对象：** [页面名称/截图描述]
- **评审时间：** [日期]
- **评审类型：** [新功能 / 迭代优化 / 竞品分析]
- **评审深度：** [快速评审 / 深度评审]

---

## 一、业务背景

### 业务目标
- **核心目标：** [服务于XX业务，达成XX指标]
- **目标用户：** [用户画像，核心诉求]
- **转化路径：** [入口 → 页面 → 出口]
- **成功定义：** [具体可衡量的目标]

---

## 二、诊断结果

### 致命问题（P0 - 阻塞级）
| 序号 | 问题 | 影响 | 建议 | 优先级 |
|------|------|------|------|--------|
| 1 | ... | 阻塞用户/业务 | ... | P0 |

### 体验问题（P1 - 重要级）
| 序号 | 问题 | 影响 | 建议 | 优先级 |
|------|------|------|------|--------|
| 1 | ... | 影响满意度 | ... | P1 |

### 优化空间（P2 - 次要级）
| 序号 | 问题 | 影响 | 建议 | 优先级 |
|------|------|------|------|--------|
| 1 | ... | 锦上添花 | ... | P2 |

### 做得好的地方
- [亮点1] - [建议如何强化]
- [亮点2] - [建议如何强化]

---

## 三、评分卡

| 维度 | 得分 | 简评 |
|------|------|------|
| 战略目标 | /10 | |
| 用户体验 | /10 | |
| 功能内容 | /10 | |
| 数据度量 | /10 | |
| 技术性能 | /10 | |
| 市场竞品 | /10 | |
| 迭代运营 | /10 | |
| **总分** | **/70** | |

**评级：** [优秀(60+) / 良好(45-59) / 一般(30-44) / 需改进(<30)]

---

## 四、方案建议

### 方案A：渐进优化（推荐快速验证）

**核心理念：**
[一句话概括]

**改动清单：**
1. [具体改动1] - [改动位置] - [预期效果]
2. [具体改动2] - [改动位置] - [预期效果]

**优点：**
- 改动小，风险低
- 可快速验证

**代价：**
- 无法解决结构性问题

---

### 方案B：结构重塑

**核心理念：**
[一句话概括]

**改动清单：**
1. [信息架构调整]
2. [交互流程优化]

**优点：**
- 解决结构性问题
- 显著提升体验

**代价：**
- 开发成本中等
- 需要用户重新学习

---

### 方案C：理想方案

**核心理念：**
[一句话概括]

**改动清单：**
1. [产品策略调整]
2. [技术架构升级]

**优点：**
- 最优的用户体验
- 长期可维护

**代价：**
- 开发成本高
- 时间周期长

---

## 五、执行计划

### 优先级排序（影响度×努力度）

| 序号 | 改动项 | 影响度 | 努力度 | 优先级 | 预期收益 |
|------|--------|--------|--------|--------|----------|
| 1 | ... | 高 | 低 | P0 | +X% |
| 2 | ... | 中 | 中 | P1 | +Y% |

### 度量指标

**核心指标（North Star）：**
- [如：页面转化率、任务完成率]

**观测指标：**
- [如：停留时长、点击率、跳出率]

**反向指标：**
- [如：报错率、放弃率]

**验证方式：**
- [ ] A/B测试
- [ ] 灰度发布
- [ ] 数据监控
- [ ] 用户调研

### 时间规划

| 阶段 | 任务 | 负责人 | 开始时间 | 结束时间 |
|------|------|--------|----------|----------|
| 设计 | ... | ... | ... | ... |

| 测试 | ... | ... | ... | ... |
| 上线 | ... | ... | ... | ... |

- **层级结构：** [几级，是否合理]
- **留白使用：** [是否有功能意义]
- **色彩使用：** [主色/辅助色数量]

### 信息密度评估
- **当前状态：** [过密/适中/过疏]
- **信息点数量：** [约X个]
- **间距情况：** [约Xpx]

### 交互状态检查
- [ ] 空状态设计
- [ ] 加载状态设计

## 七、附录
```
