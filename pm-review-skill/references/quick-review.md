# 5分钟快速评审流程

适用于初步筛选和快速评估。

---

## 快速扫描步骤

```
┌─────────────────────────────────────────────────────────┐
│  第1分钟：一眼定焦点                                      │
│  ├─ 页面核心信息是什么？                                  │
│  ├─ 用户3秒内能找到吗？                                   │
│  └─ 视觉焦点是否唯一？                                    │
├─────────────────────────────────────────────────────────┤
│  第2分钟：减法测试                                        │
│  ├─ 哪些元素可以删掉？                                    │
│  ├─ 有无重复信息？                                        │
│  └─ 装饰性元素是否过多？                                  │
├─────────────────────────────────────────────────────────┤
│  第3分钟：路径追踪                                        │
│  ├─ 用户完成核心任务需要几步？                            │
│  ├─ 是否有更短路径？                                      │
│  └─ 异常状态是否有处理？                                  │
├─────────────────────────────────────────────────────────┤
│  第4分钟：一致性检查                                      │
│  ├─ 相同功能视觉表达是否统一？                            │
│  ├─ 间距是否符合4px基准？                                 │
│  └─ 字体大小是否合理（中文≥14px）？                       │
├─────────────────────────────────────────────────────────┤
│  第5分钟：快速定性                                        │
│  ├─ 致命问题（必须修）                                    │
│  ├─ 优化空间（可以改）                                    │
│  └─ 做得好的（保留强化）                                  │
└─────────────────────────────────────────────────────────┘
```

---

## 问题分类指南

### 致命问题（P0 - 阻塞级）
- 用户无法完成核心任务
- 核心功能缺失或错误
- 严重的数据错误
- 安全/合规问题

### 体验问题（P1 - 重要级）
- 影响用户满意度但不阻塞任务
- 信息层级混乱
- 交互逻辑不清晰
- 视觉一致性差

### 优化空间（P2 - 次要级）
- 锦上添花的改进
- 性能优化
- 细节打磨
- 体验增强

## 快速评分简化标准

| 分数区间 | 评级 | 判断依据 |
|---------|------|----------|
| 8-10 | 优秀 | 做得很好，无明显问题 |
| 6-7 | 良好 | 基本达标，有小瑕疵 |
| 4-5 | 一般 | 存在明显问题，需要改进 |
| 1-3 | 差 | 严重问题，必须修复 |

## 截图快速评审特有检查

### 视觉瞬间扫描（30秒）

```
1. 一眼看过去，第一眼看到什么？ ← 核心焦点
2. 三秒内能理解页面是干嘛的吗？ ← 信息传达
3. 有没有觉得"看不过来"的感觉？ ← 信息密度
4. 颜色是不是太多了？ ← 色彩控制
5. 间距是不是太挤了？ ← 留白呼吸
```

### 交互快速检查（30秒）

```
1. 主要按钮能一眼找到吗？ ← 行动召唤
2. 有没有看到"如果...会怎样"的担忧？ ← 状态覆盖
3. 如果点错/失败了会怎样？ ← 错误处理
4. 文字太长/图片太多会怎样？ ← 边界状态
```
