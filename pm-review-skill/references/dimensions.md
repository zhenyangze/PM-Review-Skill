# 七维度评审标准

## 核心七维度速查

| 维度 | 关键问题 | 检查项 |
|------|----------|--------|
| **1. 战略目标** | 页面为什么存在？ | 业务对齐、价值主张、用户问题 |
| **2. 用户体验** | 用户能顺利完成任务吗？ | 信息层级、交互路径、状态覆盖 |
| **3. 功能内容** | 功能是否完整？ | 核心功能、内容质量、异常处理 |
| **4. 数据度量** | 效果可衡量吗？ | 埋点设计、KPI关联、A/B测试点 |
| **5. 技术性能** | 技术可行吗？ | 加载速度、兼容性、可访问性 |
| **6. 市场竞品** | 与竞品相比如何？ | 差异化、用户习惯、合规性 |
| **7. 迭代运营** | 易于维护吗？ | 模块化、扩展性、运营成本 |

---

## 评分标准

### 1. 战略目标（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 目标清晰，与业务高度对齐，价值主张明确 |
| 7-8 | 目标基本清晰，与业务关联但不够紧密 |
| 5-6 | 目标模糊，业务关联弱 |
| 1-4 | 目标缺失，与业务脱节 |

### 2. 用户体验（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 信息层级清晰，交互流畅，状态完整，异常处理完善 |
| 7-8 | 体验基本顺畅，有小瑕疵 |
| 5-6 | 存在明显体验问题，影响任务完成 |
| 1-4 | 体验差，阻塞用户完成任务 |

### 3. 功能内容（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 功能完整，内容精准，文案清晰 |
| 7-8 | 功能基本完整，内容有小问题 |
| 5-6 | 功能缺失或内容质量差 |
| 1-4 | 核心功能缺失或内容错误 |

### 4. 数据度量（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 埋点完整，KPI关联清晰，可A/B验证 |
| 7-8 | 有基本埋点，度量不够精细 |
| 5-6 | 埋点缺失，难以衡量效果 |
| 1-4 | 无度量设计 |

### 5. 技术性能（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 加载快，兼容性好，可访问性完善 |
| 7-8 | 性能可接受，有小问题 |
| 5-6 | 存在性能问题或兼容性问题 |
| 1-4 | 严重的性能或技术问题 |

### 6. 市场竞品（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 有差异化优势，符合用户习惯，合规完善 |
| 7-8 | 与竞品持平，无明显优势 |
| 5-6 | 与竞品有差距 |
| 1-4 | 明显落后于竞品 |

### 7. 迭代运营（10分）

| 分数 | 标准 |
|------|------|
| 9-10 | 模块化设计，易扩展，运营成本低 |
| 7-8 | 基本可维护，扩展性一般 |
| 5-6 | 维护成本高，扩展困难 |
| 1-4 | 难以维护，无法扩展 |

---

## 总分评级

| 总分 | 评级 | 建议 |
|------|------|------|
| 60-70 | 优秀 | 保持并强化优势 |
| 45-59 | 良好 | 针对性优化 |
| 30-44 | 一般 | 系统性改进 |
| <30 | 需改进 | 重新设计 |

## 截图评审特有维度

### 视觉层级扫描

```
层级检查清单：

□ 主焦点是否唯一？（一个页面一个视觉焦点）
□ 层级是否≤3级？（主标题 > 副标题 > 正文）
□ 留白是否有意义？（区分"呼吸空间"和"没填满"）
□ 色彩是否有功能？（主色≤1，辅助色≤2）
□ 对比度是否足够？（文字与背景对比度≥4.5:1）
```

### 信息密度分析

```
密度评估标准：

• 过密（需减负）：同一屏信息点>10个，间距<12px
• 适中（保持）：信息点5-10个，间距16-24px
• 过疏（需填充）：信息点<5个，留白>40%无功能意义
```

### 交互状态检查

```
必查状态：

□ 空状态（无数据时显示什么？）
□ 加载态（等待时反馈？）
□ 错误态（失败时引导？）
□ 成功态（完成后确认？）
□ 边界态（超长文本/大量数据？）
```

## 执行层检查清单

**每次交付前必过：**

### 战略层
- [ ] 页面目标与业务KPI关联明确
- [ ] 目标用户定义清晰
- [ ] 价值主张一句话能说清

### 体验层
- [ ] 视觉焦点唯一且明确
- [ ] 信息层级≤3级
- [ ] 核心任务路径最短
- [ ] 空/加载/错误/成功态全部覆盖

### 执行层
- [ ] 间距符合4px基准（4/8/12/16/24/32/48/64）
- [ ] 中文≥14px，英文≥13px
- [ ] 点击反馈≤50ms，loading显示阈值≥300ms
- [ ] 相同功能视觉表达统一

### 输出层
- [ ] 三套方案完整
- [ ] 优先级排序明确
- [ ] 度量指标可验证
