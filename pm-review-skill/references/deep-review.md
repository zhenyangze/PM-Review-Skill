# 30分钟深度评审流程

适用于重要页面的完整评审和方案输出。

---

## 深度评审三阶段

```
┌─────────────────────────────────────────────────────────┐
│  阶段一：诊断（10分钟）                                   │
├─────────────────────────────────────────────────────────┤
│  1. 业务对齐检查（2分钟）                                 │
│     • 页面目标与业务KPI关系                               │
│     • 目标用户是谁                                        │
│     • 核心转化路径                                        │
│                                                          │
│  2. 七维度扫描（5分钟）                                   │
│     • 战略目标 - 页面为什么存在？                         │
│     • 用户体验 - 用户能顺利完成任务吗？                   │
│     • 功能内容 - 功能是否完整？                           │
│     • 数据度量 - 效果可衡量吗？                           │
│     • 技术性能 - 技术可行吗？                             │
│     • 市场竞品 - 与竞品相比如何？                         │
│     • 迭代运营 - 易于维护吗？                             │
│                                                          │
│  3. 问题归类（3分钟）                                     │
│     • 致命缺陷（阻塞用户/业务）                           │
│     • 体验问题（影响满意度）                              │
│     • 优化空间（锦上添花）                                │
├─────────────────────────────────────────────────────────┤
│  阶段二：方案（15分钟）                                   │
├─────────────────────────────────────────────────────────┤
│  输出三套方案：                                           │
│                                                          │
│  A. 渐进优化 - 最小改动，快速验证                         │
│  B. 结构重塑 - 重新组织信息架构                           │
│  C. 理想方案 - 无限制的最优解                             │
│                                                          │
│  每套方案需包含：                                         │
│  • 核心理念（一句话）                                     │
│  • 改动清单                                              │
│  • 优点与代价                                            │
├─────────────────────────────────────────────────────────┤
│  阶段三：执行（5分钟）                                    │
├─────────────────────────────────────────────────────────┤
│  1. 优先级排序（影响度×努力度矩阵）                       │
│  2. 明确度量指标                                         │
│  3. 责任人与时间节点                                     │
└─────────────────────────────────────────────────────────┘
```

---

## 阶段一：诊断详细流程

### 1. 业务对齐检查（2分钟）

**关键问题：**
- 这个页面服务于哪个业务KPI？
- 目标用户是谁？他们的核心诉求是什么？
- 用户从哪里来？要到哪里去？（转化路径）
- 成功的定义是什么？

**输出格式：**
```markdown
## 业务背景
- **业务目标：** [服务于XX业务，达成XX指标]
- **目标用户：** [用户画像，核心诉求]
- **转化路径：** [入口 → 页面 → 出口]
- **成功定义：** [具体可衡量的目标]
```

### 2. 七维度扫描（5分钟）

每个维度快速打分（1分钟/维度）：

| 维度 | 得分 | 关键发现 |
|------|------|----------|
| 战略目标 | /10 | |
| 用户体验 | /10 | |
| 功能内容 | /10 | |
| 数据度量 | /10 | |
| 技术性能 | /10 | |
| 市场竞品 | /10 | |
| 迭代运营 | /10 | |

### 3. 问题归类（3分钟）

**按影响度×紧迫度分类：**

| 影响度 \ 紧迫度 | 高 | 中 | 低 |
|----------------|-----|-----|-----|
| 高 | P0 致命 | P1 重要 | P2 次要 |
| 中 | P1 重要 | P2 次要 | P3 可选 |
| 低 | P2 次要 | P3 可选 | P4 暂缓 |

**P0 - 致命缺陷：** 阻塞用户或业务的核心问题
**P1 - 重要问题：** 显著影响用户体验
**P2 - 次要问题：** 有改进空间
**P3/P4 - 可选/暂缓：** 锦上添花

---

## 阶段二：方案设计详细流程

### 方案A：渐进优化（最小改动，快速验证）

**定位：** 现有框架下的微调，1-2周可完成

**核心理念：**
- 一句话概括

**改动清单：**
1. [具体改动1] - [改动位置] - [预期效果]
2. [具体改动2] - [改动位置] - [预期效果]

**优点：**
- 改动小，风险低
- 可快速验证假设
- 开发成本低

**代价：**
- 无法解决结构性问题
- 优化空间有限

**适用场景：** 时间紧张、资源有限、验证假设

---

### 方案B：结构重塑（重新组织信息架构）

**定位：** 重新设计信息层级和交互流程，1个月可完成

**核心理念：**
- 一句话概括

**改动清单：**
1. [信息架构调整]
2. [交互流程优化]
3. [视觉层级重构]

**优点：**
- 解决结构性问题
- 显著提升体验
- 为后续迭代打基础

**代价：**
- 开发成本中等
- 需要用户重新学习
- 有一定风险

**适用场景：** 有明显结构性问题、有足够时间

---

### 方案C：理想方案（无限制的最优解）

**定位：** 如果可以重做，最优的解决方案，2-3个月可完成

**核心理念：**
- 一句话概括

**改动清单：**
1. [产品策略调整]
2. [技术架构升级]
3. [体验全面重构]

**优点：**
- 最优的用户体验
- 长期可维护
- 竞争力提升

**代价：**
- 开发成本高
- 时间周期长
- 风险最大

**适用场景：** 战略级产品、有长期规划、资源充足

## 阶段三：执行计划详细流程

### 1. 优先级排序（影响度×努力度矩阵）

```
高影响/低努力 → P0 立即做
高影响/高努力 → P1 计划做
低影响/低努力 → P2 填空做
低影响/高努力 → P3 不做
```

### 2. 明确度量指标

**核心指标（North Star）：**
- [主要业务指标，如转化率、完成率]

**观测指标：**
- [辅助指标，如停留时长、点击率]

**反向指标：**
- [需要监控的负面指标，如跳出率、报错率]

**验证方式：**
- [ ] A/B测试
- [ ] 灰度发布
- [ ] 数据监控
- [ ] 用户调研

### 3. 责任人与时间节点
