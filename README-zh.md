# PM 页面评审技能

[English](README.md) | [中文](README-zh.md)

一个专业的产品经理页面评审技能，适用于多种 AI 助手，通过七维度框架评估网页或设计截图的产品质量。

## 兼容的 AI 助手

本技能设计用于支持 MCP 协议的 AI 助手：

| AI 助手 | 原生支持 | 备注 |
|--------|----------|------|
| **Claude Code** | ✅ 是 | 完整 `.skill` 格式支持 |
| **Cursor** | ⚠️ 部分 | 需要适配 `.clinerules` |
| **Continue** | ⚠️ 部分 | 需要配置适配 |
| **Cline** | ⚠️ 部分 | 手动注入提示词 |
| **其他 MCP 助手** | ✅ 兼容 | 如果 MCP 服务器可用 |

## 功能特点

- **双输入支持**：支持网址评审和设计截图评审
- **两种评审模式**：快速评审（5分钟）或深度评审（30分钟）
- **七维度框架**：战略目标、用户体验、功能内容、数据度量、技术性能、市场竞品、迭代运营
- **专项深度审查**：颜色审查（六层模型）、排版审查、交互审查、无障碍审查（六层金字塔）、安全合规审查（七层模型）、性能优化审查（七层模型）
- **结构化输出**：专业评审报告，附带优先级排序建议
- **MCP 集成**：兼容 MCP 服务器增强分析能力

## 安装

### Claude Code（原生支持）

1. 将技能文件复制到技能目录：
```bash
cp pm-review-skill.skill ~/.claude/skills/
```

2. 或从解压目录安装：
```bash
mkdir -p ~/.claude/skills/pm-review-skill
cp -r pm-review-skill/* ~/.claude/skills/pm-review-skill/
```

### Cursor / Continue / 其他 AI 助手

**方式一：手动注入提示词**

添加到项目的 `.clinerules` 或提示词模板：

```
你是一个 PM 评审专家。使用以下框架：
[复制 pm-review-skill/SKILL.md 的内容]
```

**方式二：自定义指令**

配置 AI 助手的系统提示词，包含：
1. 七维度框架（来自 `dimensions.md`）
2. 评审工作流程（来自 `quick-review.md` 或 `deep-review.md`）
3. 输出模板（来自 `templates.md`）

**方式三：MCP 服务器配置**

增强功能，配置 MCP 服务器：

```json
{
  "mcpServers": {
    "web-reader": {
      "command": "npx",
      "args": ["-y", "@zengwenliang/web-reader-mcp"]
    },
    "zai-mcp": {
      "command": "npx",
      "args": ["-y", "zai-mcp-server"]
    }
  }
}
```

## 使用方法

### 触发技能

当您要求 AI 助手执行以下操作时，技能会自动激活：
- 评审页面或设计
- 分析截图
- 评估 UI/UX 设计
- 进行 PM 评审或产品评审

### 网址评审

提供网页 URL 进行功能性评审：

```
请评审 https://example.com
```

### 截图评审

提供设计图片进行视觉和交互评审：

```
请评审这张设计截图：[上传图片]
```

## 七维度评审框架

| 维度 | 核心问题 | 检查项 |
|------|----------|--------|
| **1. 战略目标** | 页面为什么存在？ | 业务对齐、价值主张、用户问题 |
| **2. 用户体验** | 用户能顺利完成任务吗？ | 信息层级、交互路径、状态覆盖 |
| **3. 功能内容** | 功能是否完整？ | 核心功能、内容质量、异常处理 |
| **4. 数据度量** | 效果可衡量吗？ | 埋点设计、KPI关联、A/B测试点 |
| **5. 技术性能** | 技术可行吗？ | 加载速度、兼容性、可访问性 |
| **6. 市场竞品** | 与竞品相比如何？ | 差异化、用户习惯、合规性 |
| **7. 迭代运营** | 易于维护吗？ | 模块化、扩展性、运营成本 |

## 评审模式

### 快速评审（5分钟）

适用于初步筛选和快速评估：
- 核心问题识别
- 优化建议
- 亮点与优势

### 深度评审（30分钟）

适用于重要页面的完整评审：
- 完整诊断报告
- 三套方案（渐进优化/结构重塑/理想方案）
- 执行计划与度量指标

## 输出结构

每次评审都包含：

1. **问题分类**：致命(P0) / 体验(P1) / 优化(P2)
2. **三套方案**：渐进优化 / 结构重塑 / 理想方案
3. **优先级排序**：影响度×努力度矩阵
4. **度量指标**：可验证的成功标准

## 评分标准

| 总分 | 评级 | 建议 |
|------|------|------|
| 60-70 | 优秀 | 保持并强化优势 |
| 45-59 | 良好 | 针对性优化 |
| 30-44 | 一般 | 系统性改进 |
| <30 | 需改进 | 重新设计 |

## AI 助手特定说明

### Claude Code
- 原生 `.skill` 格式支持
- 基于触发词自动激活技能
- MCP 服务器配置在 `~/.claude/mcp_settings.json`

### Cursor
- 在项目根目录添加 `.clinerules` 文件
- 可能需要通过自定义指令手动激活
- 通过 Cursor 设置支持 MCP

### Continue
- 添加到 Continue 的自定义指令
- 在 Continue 设置中配置 MCP 服务器
- 推荐配合 Continue Dev 扩展使用

### Cline
- 推荐手动注入提示词
- 在 Cline 设置中添加自定义指令
- MCP 服务器支持有限
